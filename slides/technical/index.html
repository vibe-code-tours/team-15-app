Dependency Injection Pattern

1

Request Arrives

POST /api/listings with JWT token

→
2

FastAPI Resolves Dependencies

Automatically injects get_current_user()

→
3

Validated User Available

current_user passed to route handler

@router.post("/listings")
def create_listing(
    body: ListingCreate,
    current_user: User = Depends(get_current_user),  # Auto-injected!
    db: Session = Depends(get_db)                      # Auto-injected!
):
    # current_user is already validated
    listing = Listing(owner_id=current_user.id, **body.model_dump())
    return listing
Benefits: Reusable • Testable • Separation of Concerns

Server vs Client Components

🖥️

Server Components

Default
When to use:
  • Static content
  • Data fetching
  • SEO-critical pages
  • Database queries
// Runs on server 
export default function Page() {
  // Can query DB directly
  return <div>Static content</div>
}
🌐

Client Components

'use client'
When to use:
  • Interactive forms
  • State management
  • Animations
  • Browser APIs
'use client' 
export function Hero() {
  const [state, setState] = useState()
  return <motion.div>...</motion.div>
}
